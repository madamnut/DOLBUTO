## Decisions
- 지형 렌더링은 현재 raymarch 방식을 기본 경로로 사용하고, F6 렌더링 전환 기능은 제거한다.
- 청크 로딩 범위는 `config/world.json`의 `loadRadius`로 조절하고, 빌드 없이 값을 바꿀 수 있게 한다.
- `loadRadius = n`이면 플레이어가 속한 청크를 중심으로 `(2n + 1) ^ 2`개의 청크를 런타임 로딩 범위로 사용한다.
- 청크는 로딩 단위로 유지하고, 런타임 렌더링 최적화와 편집 dirty 처리는 서브청크 단위를 기준으로 한다.
- 저장 파일 양식은 아직 정하지 않고, 현재 작업에서는 런타임/GPU 데이터 구조만 다룬다.
- raymarch cell은 `uint32_t` packed cell로 표현한다.
- packed cell의 하위 16비트는 block ID로 사용하고, 상위 16비트는 추후 fluid ID용으로 예약한다.
- 유체 실제 구현은 아직 진행하지 않고, 현재 packed cell의 fluid 값은 `0`으로 둔다.
- raymarch storage buffer는 GPU shader read 성능을 위해 `DEVICE_LOCAL` 메모리에 배치하고, CPU 데이터 업로드는 staging buffer를 사용한다.
- raymarch 지형 데이터는 서브청크 metadata와 dense cell buffer로 나누어 관리한다.
- 서브청크는 `EMPTY`, `UNIFORM`, `DENSE` 상태로 구분한다.
- `EMPTY` 서브청크는 cell 데이터를 저장하지 않고 raymarch 중 다음 서브청크 경계까지 건너뛴다.
- `UNIFORM` 서브청크는 cell 하나만 저장하고, `DENSE` 서브청크만 `16 x 16 x 16` cell 데이터를 가진다.
- raymarch에서 청크/서브청크 경계에 정확히 닿은 레이는 진행 방향 기준으로 경계 안쪽으로 약간 밀어 넣어 셀을 판정한다.
- 블록 texture array에만 mipmap을 적용하고, UI, 폰트, 하늘, 플레이어 텍스처에는 적용하지 않는다.
- raymarch 블록 텍스처 mip level은 자동 LOD 대신 ray hit 거리 기준 수동 `textureLod`로 선택한다.
- mip level 전환 기준은 64블록 단위로 한다.
- 게임은 Vulkan과 C++로 개발한다.
- C++ 표준은 C++20을 기준으로 한다.
- 빌드는 CMake와 Ninja를 사용한다.
- 창 생성과 입력 처리는 GLFW를 사용한다.
- Vulkan SDK는 로컬 설치본을 사용한다.
- GLFW는 Windows 바이너리 배포본을 받아서 사용하는 방식을 기준으로 한다.
- 최종 목표는 마인크래프트류 샌드박스 복셀 게임으로 잡는다.
- 첫 목표는 CMake/Ninja 프로젝트 골격, GLFW 창 생성, Vulkan instance 생성, validation layer 연결, 빈 화면 출력 준비까지로 잡는다.
- 기능을 한 파일에 몰아넣지 않고 역할에 맞게 분리해서 개발한다.
- 구현은 현재 요구사항을 만족하는 가장 단순한 형태로 진행한다.
- 파일, 클래스, 함수는 역할과 책임 기준으로 분리하되, 분리를 위한 분리는 하지 않는다.
- 새 추상화, 공용 시스템, 외부 라이브러리는 필요성이 명확할 때만 추가한다.
- 미래에 필요할 것 같은 기능은 지금 구현하지 않는다.
- 동쪽은 월드 `X+` 방향으로 정의한다.
- 위쪽은 월드 `Y+` 방향으로 정의한다.
- 북쪽은 월드 `Z+` 방향으로 정의한다.
- 블록 좌표 `(x, y, z)`는 해당 블록의 아래면 중앙 위치를 기준으로 한다.
- 초기 해 위치는 정동쪽 수평선 방향으로 둔다.
- 초기 달 위치는 해의 정반대편 방향으로 둔다.
- 해와 달 이미지는 `assets/textures/sky/Sun.png`, `assets/textures/sky/Moon.png`를 사용한다.
- 화면 중앙 크로스헤어는 `assets/textures/ui/Crosshair.png`를 사용한다.
- 하늘은 월드 위치 이동값을 반영하지 않고 카메라 회전만 반영한다.
- 디버그 텍스트는 `assets/fonts/VCR_OSD_MONO.ttf`를 사용한다.
- 디버그 텍스트는 실행 시 기본 표시 상태로 시작한다.
- FPS, 위치, 시야 방향 표시는 초당 20회 갱신한다.
- 디버그 텍스트는 흰 글씨와 검정 테두리로 표시한다.
- 우상단 디버그 정보는 버전, CPU, GPU, Vulkan API 버전, 드라이버 버전, 해상도 순서로 표시한다.
- 해상도 표시는 `RESOLUTION: 가로 x 세로` 형식으로 표시한다.
- 청크 크기는 `X 16 * Y 512 * Z 16`으로 정의한다.
- 청크는 로딩과 언로딩 단위로 사용한다.
- 청크 하나는 Y축 방향으로 `16 * 16 * 16` 크기의 서브청크 32개로 구성한다.
- 서브청크는 메싱 단위로 사용한다.
- 각 블록 셀은 블록 ID와 유체 ID를 함께 가진다.
- 블록 ID와 유체 ID는 각각 65536가지 값을 표현할 수 있도록 `uint16` 범위를 사용한다.
- 블록 ID `0`은 `air`로 예약하며 JSON 블록 정의에 명시한다.
- 초기 블록 ID는 `1 rock`, `2 grass`, `3 dirt`, `4 sand`, `5 sandstone`, `6 mud`, `7 clay`, `8 trunk`, `9 leaves`, `10000 plant`, `65535 bedrock`으로 정의한다.
- 블록 정의 JSON은 `id`, `name`, `renderType`, `textures`를 명시한다.
- 블록 텍스처 값은 확장자 없는 이름으로 작성하고, 실제 파일은 `assets/textures/block/{name}.png`로 해석한다.
- 유체 데이터는 청크 데이터 구조에 포함하지만, 초기 구현에서는 렌더링, 메싱, 시뮬레이션에 사용하지 않는다.
- 유체 데이터의 상위 9비트는 유체 종류, 하위 7비트는 유체량으로 사용한다.
- 유체 없음은 `type = 0`, `amount = 0`으로 표현한다.
- 유체 있음은 `type = 1~511`, `amount = 1~100`으로 표현한다.
- `amount = 100`은 가득 찬 상태를 의미한다.
- `amount = 101~127`은 예약값으로 사용하지 않는다.
- 블록 편집 시 변경 블록이 속한 서브청크를 dirty 처리하고, 경계 블록이면 인접 서브청크도 dirty 처리한다.
