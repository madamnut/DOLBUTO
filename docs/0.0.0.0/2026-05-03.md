## Decisions
- 플레이어 이동 파라미터는 실행 시작 시 `config/world.json`의 `player` 오브젝트에서 한 번 읽고, 값이 없거나 범위를 벗어나면 코드 기본값을 사용한다.
- 카메라 회전은 프레임 단위 입력을 유지하고, 플레이어 위치만 20Hz 물리 위치 사이를 보간해 렌더링한다.
- ground 모드 수직 이동은 현재 속도로 먼저 이동/충돌 처리한 뒤 중력을 적용한다.
- `groundMoveSpeed`는 ground 모드에서 입력 방향에 바로 적용되는 기준 걷기속도로 해석한다.
- 지형 설정은 `terrain.domainWarp`를 먼저 적용한 뒤 `terrain.baseNoise`로 높이 노이즈를 계산하는 순서로 둔다.
- 블럭 편집은 우선 청크 전체 재메싱으로 처리하고, 편집 위치가 청크 경계에 닿으면 인접 청크도 같이 재메싱한다.
- 블럭 선택 판정은 3인칭 시점에서도 1인칭 눈 위치와 시야 방향 기준으로 하고, 테두리 렌더링만 현재 렌더 카메라에서 보이도록 한다.
- 플레이어 `POS`는 발밑 중앙으로 유지하고, 충돌 박스는 X/Z 반폭 0.3, Y 높이 1.75로 판정한다.
- 플레이어 충돌은 현재 로딩된 청크 데이터와 `blocks.json`의 `collision` 필드를 기준으로 한다.
- 빠른 이동 중 블럭을 관통하지 않도록 이동량을 작은 step으로 쪼개고 X/Y/Z 축을 나눠 적용한다.
- `faceOcclusion`은 우선 블럭 6면 공통 값으로 둔다.
- `alphaMode`는 `opaque`, `cutout`, `blend`를 정의하되, 현재 렌더링 경로에서는 opaque/cutout 블럭 정의 준비까지만 한다.
- 지형 노이즈의 fractal 타입은 일단 FBm으로 고정하고, 수치 파라미터만 JSON으로 노출한다.
- `terrain.baseNoise.octaveCount`는 안전하게 1~16 범위로 제한한다.
- 도메인 워핑은 기본 비활성화 상태로 두고, 활성화 시 같은 4D 토러스 좌표에서 만든 warp 노이즈를 좌표에 더해 타일링을 유지한다.
- `terrain.domainWarp.octaveCount`도 1~16 범위로 제한한다.
- 블럭 텍스처는 개별 텍스처를 바인딩하지 않고, `blocks.json`에 등장하는 모든 텍스처를 하나의 텍스처 어레이에 올리는 방식으로 한다.
- `plant` 같은 cross 타입도 지금은 생성하지 않지만, 텍스처 레이어에는 포함해 나중에 렌더 타입만 따로 붙일 수 있게 둔다.
- 그리디 메싱 병합 조건에는 AO signature뿐 아니라 texture layer도 포함한다.
- 밉맵 선택은 화면 자동 LOD 대신 카메라와 프래그먼트 월드 위치의 거리 기준으로 한다.
- 현재 거리 구간은 `0~31: 32x32`, `32~63: 16x16`, `64~95: 8x8`, `96~127: 4x4`, `128~159: 2x2`, `160 이상: 1x1`로 둔다.
- 프러스텀 컬링은 일단 CPU에서 단순 AABB 판정으로 처리한다.
- 컬링 단위는 청크 전체가 아니라 현재 GPU 버퍼 단위와 맞는 서브청크로 둔다.
- near/far plane 값은 지형 렌더링과 컬링 판정에서 같은 상수를 사용한다.
- 경계 1칸 데이터는 저장용 청크 데이터에 포함하지 않고, 메싱 잡 안에서만 생성한 뒤 폐기한다.
- ambient occlusion은 런타임 화면 후처리가 아니라 메싱 시점에 계산하는 정적 vertex AO로 둔다.
- AO 때문에 그리디 메싱을 완전히 끄지는 않고, AO가 안전하게 같은 영역만 병합한다.
