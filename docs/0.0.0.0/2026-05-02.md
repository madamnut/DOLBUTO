## Decisions
- 저장/로딩 단위는 청크, 메싱 단위는 서브청크, GPU 업로드는 메인 스레드에서 제한 처리하는 방향으로 간다.
- 첫 잡 시스템은 dependency graph 없이 완료 큐를 보고 다음 단계 잡을 넣는 단순한 단계식 구조로 둔다.
- 청크 데이터가 준비된 뒤의 메쉬 생성 잡은 남은 데이터 생성 잡보다 우선 처리한다.
- 로딩 우선순위는 플레이어 시야 방향을 고려하지 않고, 중심 2x2 청크 기준 정사각형 링 순서만 사용한다.
- 월드 로딩 메싱은 청크 단위로 처리하고, 블럭 설치/파괴 같은 편집 발생 시에만 서브청크 단위 재메싱을 적용하는 방향으로 간다.
- 빠른 이동으로 필요 없어진 잡은 시작 전 generation 검사로 건너뛰고, 이미 실행 중인 잡은 완료 후 메인 스레드에서 폐기한다.
- 프로그램 종료 시에는 남은 지형 잡 결과를 보존하지 않고 폐기한다.
- 월드 렌더링은 레이마칭을 사용하지 않고 메쉬 기반 렌더링으로 진행한다.
- 기존 F4 와이어프레임 토글은 메쉬 지형 파이프라인에 연결해서 유지한다.
- GPU가 아직 참조할 수 있는 지형 버퍼는 언로드 시점에 바로 destroy하지 않고, in-flight 프레임이 지나간 뒤 파괴한다.

- FastNoise2와 FastSIMD는 모두 `external/` 아래 vendor 소스로 둔다.
- 월드 heightmap noise 생성은 `GenTileable2D`가 아니라 `GenPositionArray4D` batch API를 사용한다.
- 지형 기본 노드는 `Simplex`를 source로 둔 `FractalFBm`으로 구성하고, 초기값은 `octaves=4`, `lacunarity=2.0`, `gain=0.5`로 둔다.
- 토러스 좌표 변환은 청크 단위로 X 16개, Z 16개의 sin/cos 값을 먼저 계산한 뒤 16x16 좌표 배열로 조합한다.
- 월드 height 변환은 C++ 하드코딩 곡선 대신 `assets/data/world`의 height LUT binary를 사용하는 방향으로 둔다.
- height LUT는 noise `-2.0~2.0` 범위를 X축 기준 1024개로 나눈다.
- FastNoise2는 프로젝트 CMake에 연결된 상태로 사용한다.
- FastNoise2는 단일 샘플 호출보다 batch API를 사용해 청크 단위 높이값을 생성하는 방향을 우선한다.
- 기본 월드 heightmap은 4D simplex noise를 2D torus 좌표로 샘플링하는 방식으로 타일링한다.
- 월드 heightmap 타일 주기는 `65536` 블럭으로 둔다.

- FastSIMD는 CMake에 들어갈 때 반드시 정규화된 절대 경로로 넘긴다.

- 레이마칭을 제거하면서 내부 렌더 해상도/업스케일 패스는 더 이상 사용하지 않는다.
