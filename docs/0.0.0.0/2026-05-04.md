## Work Log

- 블럭 편집 후 리메시가 청크 전체 32개 서브청크를 다시 만들던 흐름을 수정했다.
- 편집된 블럭이 포함된 서브청크와 경계 영향을 받는 인접 서브청크/청크만 즉시 리메시하도록 변경했다.
- 블럭 편집 리메시는 청크 로딩용 `18 x 512 x 18` 메싱 입력을 만들지 않고, 편집 서브청크 주변 `18 x 18 x 18` 임시 입력만 만들도록 분리했다.
- 청크 로딩 순서를 링 순회 방식에서 중심 2x2 청크 그룹 기준 거리순 오프셋 테이블 방식으로 변경했다.
- 블럭 정의 JSON과 런타임 블럭 정의 테이블에 방향성 여부를 나타내는 `directional` 필드를 추가했다.
- 전역 좌표 기반 `worldRandom8` 해시 랜덤을 추가했다.
- 각 월드 컬럼의 맨 아래 bedrock 두께를 `worldRandom8` 기반 1~4블럭으로 생성하도록 변경했다.
- `directional`이 false인 블럭의 윗면 텍스처는 `worldRandom8` 기반 4방향 회전을 적용하도록 변경했다.
- 지형 생성 시 가장 윗블럭이 grass인 컬럼에는 70% 확률로 그 위에 plant 블럭을 배치하도록 추가했다.
- `cross` 렌더 타입 블럭을 X자 형태의 양면 쿼드로 메싱하도록 추가했다.
- terrain fragment shader에서 알파값 0.5 미만 픽셀을 discard해 plant/leaves 같은 cutout 텍스처가 뚫려 보이도록 했다.
- 블럭 정의 JSON과 terrain vertex에 `mipDistanceScale`을 추가해 블럭별 밉맵 전환 거리를 조정할 수 있도록 변경했다.
- 블럭 텍스처 어레이 로딩 시 `assets/textures/block/mip/<텍스처명>_mipN.png` 수동 mip 파일이 있으면 해당 mip level을 우선 사용하도록 추가했다.
- 블럭 텍스처의 누락된 mip level은 실행 중 직전 mip level에서 생성하고 `assets/textures/block/mip/` 폴더에 PNG로 저장하도록 추가했다.
- 생성된 mip PNG 저장은 직접 작성한 PNG writer 대신 `external/stb/stb_image_write.h`의 `stbi_write_png`를 사용하도록 변경했다.
- 밉맵 전환 기준을 `mipDistanceScale` 1.0일 때 64블럭 단위로 변경하고, 모든 블럭 텍스처의 `mipDistanceScale`을 1.0으로 맞췄다.

## Decisions
- 블럭 상호작용 레이는 렌더링에서 실제로 보이는 시야 방향과 동일하게 `camera.forward()`의 Y 성분을 반전한 방향을 사용한다.
- 이동 입력 계산에 쓰는 카메라 방향은 기존 값을 유지하고, 블럭 상호작용과 3인칭 카메라 위치 계산에만 화면 기준 방향을 사용한다.
- 청크 로딩은 기존처럼 청크 전체 메시 잡을 사용하지만, 블럭 편집 리메시는 서브청크 단위 GPU 버퍼만 교체한다.
- 편집 위치가 X/Z 청크 경계나 Y 서브청크 경계에 닿으면 AO/면 컬링 갱신을 위해 해당 경계의 인접 단위도 함께 리메시한다.
- 청크 로딩 메싱은 월드 생성용 heightmap 경계 데이터를 사용하고, 블럭 편집 메싱은 현재 런타임 청크 데이터에서 작은 서브청크 경계 버퍼를 만들어 사용한다.
- 청크 로딩 오프셋 테이블은 로딩 지름이 바뀔 때만 다시 만들고, 각 오프셋은 중심 2x2 청크 그룹까지의 거리 제곱 기준으로 정렬한다.
- 같은 거리의 청크들은 별도 방향 우선순위를 두지 않고 테이블 생성 순서에 맡긴다.
- `directional`은 블럭 타입이 방향성을 지원하는지 여부만 나타내며, 개별 블럭 인스턴스의 실제 방향값 저장은 아직 추가하지 않는다.
- 완전 랜덤 값은 저장하지 않고 월드 좌표와 salt에서 다시 계산한다.
- 비방향성 블럭의 윗면 텍스처 회전은 내부적으로 0~3 값으로 계산해 0도, 90도, 180도, 270도 회전으로 사용한다.
- 윗면 회전값이 다른 면끼리는 그리디 메싱에서 병합하지 않는다.
- plant 배치는 grass 표면 위 1블럭 높이에 두며, 좌표 기반 `worldRandom8`로 약 70% 확률을 판정한다.
- cross 타입은 그리디 메싱 대상이 아니며 개별 블럭마다 두 장의 교차 평면을 양면으로 생성한다.
- `mipDistanceScale`은 0.0이면 밉맵을 끄고, 1.0이면 64블럭마다 밉맵을 전환하며, 2.0이면 128블럭마다 전환하는 식으로 해석한다.
- 현재 모든 블럭 텍스처는 `mipDistanceScale`을 1.0으로 둔다.
- `mipDistanceScale`이 다른 cube 면끼리는 그리디 메싱에서 병합하지 않는다.
- 수동 mip 파일은 `assets/textures/block/mip/rock_mip1.png`, `assets/textures/block/mip/rock_mip2.png` 같은 이름으로 둔다.
- 수동 mip level 사이에 빠진 level은 직전 mip level에서 자동 생성하고 같은 mip 폴더에 저장한다.
- 이미지 읽기는 `stb_image`, 이미지 쓰기는 `stb_image_write`를 사용하며, implementation은 `src/third_party`의 별도 cpp 파일에서 한 번만 정의한다.
